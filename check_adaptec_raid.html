NAME

check_adaptec_raid - Nagios/Icinga Plugin to check Adaptec raid-controllers.

SYNOPSIS

Example:

        ./check_adaptec_raid -Tw 70 -Tc 80 -c 1
        AACRAID CRITICAL (Ctrl #1): [Log. device failed/degraded,
        Log. device critical, 1 log. device(s) failed]

VERSION

This document describes check_adaptec_raid version 0.1.

DESCRIPTION

check_adaptec_raid is a Nagios/Icinga Plugin to check Adaptec raid-controllers for errors/warnings. This plugin makes heavy use of 'arcconf' which can be obtained from the Adaptec website. It checks the controller, the physical devices and the logical devices for errors or warnings (for more detailed information about what is being monitored see METHODS).

DEPENDENCIES

- This plugin requires a running Nagios or Icinga. (either local or NRPE)

- sudo

- arcconf (Can be obtained from the Adaptec Homepage)

- The following Perl Modules:

        use strict;
        use warnings;
        use Getopt::Long qw(:config no_ignore_case);

METHODS

myStatus

Sets the correct exit code for each function. Can only increment the exit code (if necessary) since the exit code may not change from Warning to OK.

displayUsage

Explains the usage of the plugin and which parameters are available.

displayHelp

Prints a short help text and quick information where to find additional help.

displayVersion

Prints the plugin version and the arcconf version.

getThresholds

Uses the -Tw --temperature-warn and -Tc --temperature-critical parameters and parses them for getControllerCfg

Uses correct Nagios threshold implementation: http://nagiosplug.sourceforge.net/developer-guidelines.html#THRESHOLDFORMAT

Returns a temperature range (array) in or out which a temperature should be

Array content: ("in" or "out", range from, range to)

Examples

Generate an alert if x...
-Tw 10< 0 or > 10, (outside the range of {0 .. 10})
-Tw 10:< 10, (outside {10 .. inf})
-Tw ~:10> 10, (outside the range of {-inf .. 10})
-Tw 10:20< 10 or > 20, (outside the range of {10 .. 20})
-Tw @10:20>= 10 and <= 20, (inside the range of {10 .. 20})

getControllerCfg

Checks the Adaptec raid-controller for controller errors. (uses arcconf GETCONFIG 1 AD)

- Controller status

- Defunctional disks

- Logical devices which have failed/are degraded

- Temperature

- Status of ZMM if present

The controller temperature can be set to a warning (parameter: -Tw|--temperature-warn) and a critical level (parameter: -Tc|--temperature-critical). See getThresholds for more information. If no ZMM Module is present the check can be disabled (--zmm 0).

getPhysDevCfg

Returns information about physical devices attached to the Adaptec raid-controller:

- Disk status

- S.M.A.R.T. status

- S.M.A.R.T. warnings

- Low-RPM warnings

- Failed disk segments

getLogDevCfg

Returns information about logical devices:

- General logical devices status

- Failed stripes
